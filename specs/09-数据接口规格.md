| 项 | 值 |

| 模块编号 | M1-MuseumGuide |

---

> **说明**: 本项目为纯前端静态应用，无后端 API。本文档定义前端数据层的接口契约，即 `src/data/index.ts` 导出的查询函数和数据导出。

## 1. 数据层接口总览

| 接口 | 类型 | 返回值 | 说明 |
|------|------|--------|------|
| `museums` | 常量导出 | `Museum[]` | 全部博物馆数据数组 |
| `getMuseumById(id)` | 函数 | `Museum \| undefined` | 按 ID 查询单个博物馆 |
| `getExhibitById(museumId, exhibitId)` | 函数 | `Exhibit \| undefined` | 按博物馆 ID 和展品 ID 查询展品 |

## 2. 接口详细规格

### 2.1 museums 常量

```typescript
export const museums: Museum[]
```

**数据来源**: 静态导入自 `museums/forbidden-city.ts`、`museums/uffizi.ts`、`museums/british-museum.ts`

**数据保证**:
- MUST 包含 3 个博物馆对象
- 每个博物馆 MUST 有唯一 `id` 字段
- `id` 值: `"forbidden-city"` | `"uffizi"` | `"british-museum"`

**示例**:
```typescript
[
  { id: "forbidden-city", name: "故宫博物院", ... },
  { id: "uffizi", name: "乌菲齐美术馆", ... },
  { id: "british-museum", name: "大英博物馆", ... }
]
```

### 2.2 getMuseumById

```typescript
export function getMuseumById(id: string): Museum | undefined
```

| 参数 | 类型 | 必填 | 约束 |
|------|------|------|------|
| id | string | 是 | 博物馆 ID |

**行为规格**:
- Given 有效 ID（如 `"forbidden-city"`），MUST 返回对应 `Museum` 对象
- Given 无效 ID（如 `"louvre"`），MUST 返回 `undefined`
- 查找方式: `museums.find(m => m.id === id)`

**示例**:
```typescript
getMuseumById("uffizi")
// → { id: "uffizi", name: "乌菲齐美术馆", ... }

getMuseumById("nonexistent")
// → undefined
```

### 2.3 getExhibitById

```typescript
export function getExhibitById(museumId: string, exhibitId: string): Exhibit | undefined
```

| 参数 | 类型 | 必填 | 约束 |
|------|------|------|------|
| museumId | string | 是 | 博物馆 ID |
| exhibitId | string | 是 | 展品 ID |

**行为规格**:
- 先通过 `getMuseumById` 查找博物馆
- 在 `featuredExhibits` 和 `hiddenGems` 中搜索展品
- Given 有效的博物馆 ID 和展品 ID，MUST 返回对应 `Exhibit` 对象
- Given 无效的博物馆 ID 或展品 ID，MUST 返回 `undefined`

**示例**:
```typescript
getExhibitById("forbidden-city", "qingming-scroll")
// → { id: "qingming-scroll", name: "清明上河图", ... }

getExhibitById("forbidden-city", "nonexistent")
// → undefined
```

## 3. Context 接口

### 3.1 GuidePreferenceContext

```typescript
type GuideMode = 'popular' | 'enthusiast';

interface GuidePreferenceContextType {
  mode: GuideMode;
  setMode: (mode: GuideMode) => void;
}
```

**Provider**: `GuidePreferenceProvider`

**持久化契约**:
- MUST 在 `setMode` 调用时写入 `localStorage`
- Key: `"guide-preference"`
- Value: `"popular"` | `"enthusiast"`
- MUST 在初始化时读取 localStorage，若无记录则默认 `"popular"`

**使用方式**:
```typescript
const { mode, setMode } = useGuidePreference();
```

## 4. Hook 接口

### 4.1 useSnapScroll

```typescript
function useSnapScroll(totalPages: number): {
  containerRef: React.RefObject<HTMLDivElement>;
  currentPage: number;
  scrollToPage: (index: number) => void;
}
```

| 参数 | 类型 | 必填 | 约束 |
|------|------|------|------|
| totalPages | number | 是 | 总页数，MUST > 0 |

**返回值契约**:
- `containerRef`: MUST 绑定到 `.snap-container` 元素
- `currentPage`: 0-based 当前可见页码，MUST 在 `[0, totalPages-1]` 范围内
- `scrollToPage(index)`: 调用后 MUST 平滑滚动到指定页面

**IntersectionObserver 配置**:
- `threshold: 0.5` — 页面可见 50% 时判定为当前页
- `root: containerRef.current` — 限定在 snap 容器内观察
