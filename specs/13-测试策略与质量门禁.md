## 一、测试分层

| 层级 | 类型 | 工具 | 覆盖范围 |
|------|------|------|---------|
| L1 | 静态分析 | TypeScript + ESLint | 类型安全 + 代码风格 |
| L2 | 组件测试 | Vitest + React Testing Library（规划中）| 组件渲染和交互 |
| L3 | 视觉回归 | Chromatic / Percy（规划中）| UI 视觉一致性 |
| L4 | E2E 测试 | Playwright（规划中）| 端到端用户流程 |
| L5 | 性能测试 | Lighthouse CI | Web Vitals 指标 |
| L6 | 兼容性测试 | iPhone 真机 + Safari 模拟器 | 设备兼容性 |

## 二、测试用例拆分原则

1. **每个 P0 用户故事至少 3 个测试用例**
2. **每个组件至少 1 个渲染测试**
3. **每个交互行为至少 1 个交互测试**
4. **每个降级场景至少 1 个边界测试**

## 四、质量门禁

| 门禁 ID | 检查项 | 通过条件 | 触发时机 |
|---------|--------|---------|---------|
| G-LINT | ESLint 检查 | 零 error，warning ≤ 10 | 每次提交 |
| G-TYPE | TypeScript 编译 | `tsc --noEmit` 零错误 | 每次构建 |
| G-BUILD | 生产构建 | `npm run build` 成功 | 每次部署 |
| G-PERF | Lighthouse 性能 | Performance ≥ 90 | 每次部署 |
| G-SIZE | 产物体积 | gzip ≤ 500KB（不含图片）| 每次构建 |
