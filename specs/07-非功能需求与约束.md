|---|---|
| 模块编号 | M1-MuseumGuide |

## 1. 性能需求

### 1.1 加载性能 SLO

| 指标 | 目标值 | 度量条件 | 工具 |
|------|--------|---------|------|
| FCP (First Contentful Paint) | ≤ 1.5s | 4G 网络 / iPhone 标准版 | Lighthouse |
| LCP (Largest Contentful Paint) | ≤ 2.5s | 4G 网络 / 首屏英雄图 | Lighthouse |
| TTI (Time to Interactive) | ≤ 3.0s | 4G 网络 | Lighthouse |
| CLS (Cumulative Layout Shift) | ≤ 0.1 | 页面加载全程 | Lighthouse |
| 构建产物体积 | ≤ 500KB (gzip) | 不含图片资源 | Vite Bundle Analyzer |

### 1.2 运行时性能

| 指标 | 目标值 | 场景 |
|------|--------|------|
| 滑动帧率 | ≥ 55fps | Snap 容器垂直滑动 |
| 动画帧率 | ≥ 55fps | Framer Motion 过渡 |
| 页面切换延迟 | ≤ 100ms | 路由跳转 |
| 页码指示器响应 | ≤ 50ms | 点击/长按触发 |

### 1.3 图片加载

| 策略 | 实现方式 |
|------|---------|
| 懒加载 | `loading="lazy"` HTML 原生属性 |
| 异步解码 | `decoding="async"` |
| 骨架屏 | shimmer 动画占位 |
| 渐显过渡 | `opacity: 0 → 1` 300ms transition |
| 分辨率适配 | 1280px 宽度缩略图（适配 3x 像素比）|

## 2. 兼容性需求

### 2.1 设备兼容

| 设备 | 屏幕宽度 | 像素比 | 字号 | 状态 |
|------|---------|--------|------|------|
| iPhone SE | 375px | 2x | 15px | MUST |
| iPhone 15/16 | 390px | 3x | 16px | MUST |
| iPhone 15/16 Plus | 428px | 3x | 16px | MUST |
| iPhone 15/16 Pro Max | 430px | 3x | 16.5px | MUST |

### 2.2 浏览器兼容

| 浏览器 | 版本 | 优先级 |
|--------|------|--------|
| Safari (iOS) | ≥ 15 | MUST |
| Safari PWA (standalone) | ≥ 15 | MUST |
| Chrome (iOS) | ≥ 100 | SHOULD |
| 桌面浏览器 | 不适配 | NOT IN SCOPE |

### 2.3 CSS 特性兼容

| 特性 | 前缀/降级 |
|------|---------|
| `backdrop-filter` | MUST 提供 `-webkit-backdrop-filter` |
| `dvh` 单位 | MUST 使用，不提供 `vh` 降级（iOS 15+ 已支持）|
| `env(safe-area-inset-*)` | MUST 使用 `viewport-fit=cover` 配合 |
| `scroll-snap-type` | 无需前缀（Safari 14.1+ 原生支持）|
| `will-change` | MUST 限制使用范围，避免内存消耗 |

## 3. 可用性需求

| 编号 | 需求 | 说明 |
|------|------|------|
| ACC-01 | 触控目标 ≥ 44px | 符合 Apple Human Interface Guidelines |
| ACC-02 | 颜色对比度 ≥ 4.5:1 | 正文文字与背景（深棕 #3D2C23 on 米白 #FAF8F5）|
| ACC-03 | 交互即时反馈 | 按钮 active 状态提供视觉/触觉反馈 |
| ACC-04 | 滚动方向一致 | 全应用统一垂直滑动方向 |
| ACC-05 | 导航可预期 | 所有子页面提供返回按钮 |

## 4. 可观测性

| 维度 | 当前状态 | 后续计划 |
|------|---------|---------|
| 错误监控 | 无 | 二期接入 Sentry |
| 访问统计 | 无 | 二期接入 Vercel Analytics |
| 性能监控 | 手动 Lighthouse | 二期接入 Web Vitals |

## 5. 安全约束

| 编号 | 约束 | 说明 |
|------|------|------|
| SEC-01 | 无用户输入 | 纯展示型应用，无 XSS 风险入口 |
| SEC-02 | 外部资源 HTTPS | 所有图片源 MUST 使用 HTTPS |
| SEC-03 | localStorage 无敏感信息 | 仅存储版本偏好（非 PII）|
| SEC-04 | CSP 头 | Vercel 默认 CSP 策略 |

## 6. 数据约束

| 约束 | 说明 |
|------|------|
| 数据只读 | 所有博物馆/展品数据为静态内置，运行时不可修改 |
| 无网络请求 | 数据层无 API 调用，仅本地导入 |
| 图片外链 | 图片通过 Unsplash/Wikimedia URL 引用，依赖外部 CDN |
| 本地存储上限 | localStorage 仅存 1 个 key（guide-preference），≤ 20 bytes |

## 7. 部署约束

| 约束 | 说明 |
|------|------|
| 静态托管 | MUST 为纯静态产物，无服务端渲染 |
| 构建产物 | Vite 输出到 `dist/` 目录 |
| 路由模式 | Hash Router，无需服务端 rewrite 规则 |
| CDN 缓存 | Vercel 自动处理，带 hash 的资源文件长缓存 |

## 8. 方向与布局约束

| 约束 | 实现 |
|------|------|
| 竖屏锁定 | manifest.json `orientation: portrait` |
| 横屏警告 | CSS `@media (orientation: landscape) and (max-height: 600px)` |
| 视口设置 | `viewport-fit=cover` + `width=device-width` |
| 缩放禁止 | `user-scalable=no` + `maximum-scale=1.0` |
