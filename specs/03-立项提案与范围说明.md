## 1. 背景与问题（痛点表）

| 编号 | 痛点 | 影响范围 | 严重程度 |
|------|------|---------|---------|
| P-01 | 博物馆攻略信息碎片化，散落在多个平台 | 所有用户 | 高 |
| P-02 | 传统网页/App 移动端浏览体验差 | 移动端用户 | 高 |
| P-03 | 展品介绍内容单一，无法满足不同深度需求 | 艺术爱好者 | 中 |
| P-04 | 缺乏智能化的游览路线推荐 | 现场游客 | 高 |
| P-05 | 现有方案不支持离线/添加主屏幕 | 出境游客 | 中 |

## 2. 提案概要

构建一款面向 iPhone 用户的文艺型博物馆导览 H5 WebApp「博物馆漫步」。采用类小红书/抖音的垂直全屏吸附滑动交互，提供故宫博物院、乌菲齐美术馆、大英博物馆三家世界级博物馆的完整结构化导览。支持「提前做攻略」和「到现场导览」双场景，提供大众版/爱好者版双版本展品深度内容，配合智能路线推荐系统。纯前端静态架构，Vercel 部署上线。

## 3. 范围边界

### 3.1 范围内（MUST DO）

| 编号 | 功能 | 说明 |
|------|------|------|
| IN-01 | 三个博物馆完整导览数据 | 故宫、乌菲齐、大英，含展品/路线/楼层 |
| IN-02 | 垂直全屏吸附滑动交互 | CSS Scroll Snap 实现 |
| IN-03 | 双场景模式 | 提前做攻略 + 到现场导览 |
| IN-04 | 智能路线推荐 | 按时间选择推荐不同路线方案 |
| IN-05 | 双版本展品内容 | 大众版（通俗）+ 爱好者版（学术）|
| IN-06 | iPhone 全系适配 | SE ~ Pro Max，含安全区域适配 |
| IN-07 | PWA 基础支持 | manifest.json、添加到主屏幕、竖屏锁定 |
| IN-08 | Vercel 部署上线 | 自动化构建部署、全球 CDN |

## 5. 风险评估

| 编号 | 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|------|---------|
| R-01 | 外部图片源 CORS 问题 | 高 | 图片无法加载 | 已解决：迁移至 Unsplash CDN |
| R-02 | iOS Safari CSS 兼容性 | 中 | 样式异常 | -webkit 前缀 + 真机测试 |
| R-03 | 动态视口高度计算差异 | 中 | 布局错位 | 使用 dvh 单位 |
| R-04 | 内容数据量大导致首屏慢 | 低 | 加载体验差 | 懒加载 + Tree-shaking |
| R-05 | Hash Router SEO 不友好 | 低 | 搜索引擎无法索引 | 当前可接受，后续考虑 SSR |

## 6. 项目约束

| 约束类型 | 说明 |
|---------|------|
| **技术约束** | 纯前端静态应用，无后端服务 |
| **设备约束** | 一期仅适配 iPhone（375-430px），竖屏方向 |
| **内容约束** | 三个博物馆，展品数据由 AI 辅助生成 |
| **部署约束** | Vercel 免费方案 |
| **语言约束** | 仅中文 |
