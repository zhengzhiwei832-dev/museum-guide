## 1. 文档体系说明

本文档体系为「博物馆漫步」项目的完整 Spec 规格文档集，覆盖需求采集、产品设计、技术架构、数据模型、测试策略到需求追踪的全生命周期。

### 1.1 适用范围

- 项目代号：museum-guide
- 产品形态：面向 iPhone 用户的文艺型博物馆导览 H5 WebApp
- 技术形态：纯前端静态应用（React + Vite + TypeScript）
- 部署形态：Vercel 静态托管

### 1.2 写作原则

| 原则 | 说明 |
|------|------|
| **单一真相** | 数据类型以 `10` 为准，前端接口以 `09` 为准 |
| **先行为后实现** | 先定义 `05` 用户故事，再写 `06/09/10` |
| **可验证** | 所有 MUST 条目必须能被测试验证 |
| **不混层** | PRD 不写 CSS，功能规格不写数据结构 |
| **无歧义** | 禁止「可选/建议/大概/尽量」等模糊表述 |

### 1.3 规范词定义（RFC 风格）

| 词 | 含义 | 使用场景 |
|---|------|---------|
| **MUST** | 必须，违反即缺陷 | 核心功能、安全约束 |
| **SHOULD** | 建议，不满足需说明理由 | 性能优化、最佳实践 |
| **MAY** | 可选，不影响基线验收 | 增强功能 |
| **MUST NOT** | 严禁 | 安全风险、数据泄露 |
