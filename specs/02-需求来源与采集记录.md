---

## 1. 需求背景

### 1.1 市场痛点

| 编号 | 痛点描述 | 来源 |
|------|---------|------|
| P-01 | 博物馆官方 App 功能臃肿，启动慢，用户体验差 | 用户调研 |
| P-02 | 现有攻略散落在小红书、马蜂窝等平台，信息碎片化，缺乏结构化整合 | 竞品分析 |
| P-03 | 到现场后不知道看什么、怎么规划路线，浪费大量时间排队和走冤枉路 | 用户调研 |
| P-04 | 展品介绍要么太专业看不懂，要么太浅薄没有深度，缺乏分层内容 | 内容分析 |
| P-05 | 移动端浏览体验差，传统网页不适合单手竖屏操作 | 体验评估 |

### 1.2 目标用户画像

| 用户角色 | 特征 | 核心需求 |
|---------|------|---------|
| **文艺青年** | 20-35岁，热爱文化艺术，经常逛博物馆 | 深度内容、沉浸式体验 |
| **旅行规划者** | 出行前做功课，追求高效游览 | 结构化攻略、路线推荐 |
| **现场游客** | 已到博物馆，需要即时导览 | 快速查找展品、位置导引 |
| **艺术爱好者** | 对艺术史有一定了解，追求学术深度 | 专业解读、学术分析 |

## 2. 需求采集记录

### S-01 产品灵感来源

| 时间 | 来源 | 内容 |
|------|------|------|
| 2025-03 | 产品设计 | 参考小红书/抖音的垂直全屏滑动交互模式，将博物馆导览体验化 |
| 2025-03 | 交互设计 | 移动端优先，iPhone 专属适配，追求原生 App 级体验 |
| 2025-03 | 内容策略 | 双版本内容体系——大众版（通俗易懂）+ 爱好者版（学术深度）|

### S-02 核心需求提炼

| 需求编号 | 需求描述 | 优先级 | 来源 |
|---------|---------|--------|------|
| REQ-01 | 提供三个世界级博物馆的结构化导览内容 | P0 | 产品规划 |
| REQ-02 | 支持「提前做攻略」和「到现场导览」两种使用场景 | P0 | 用户调研 |
| REQ-03 | 垂直全屏吸附滑动的沉浸式浏览体验 | P0 | 交互设计 |
| REQ-04 | 智能路线推荐（按时间选择不同路线方案）| P0 | 用户调研 |
| REQ-05 | 展品双版本深度内容（大众版/爱好者版）| P0 | 内容策略 |
| REQ-06 | iPhone 全系机型完美适配 | P0 | 技术要求 |
| REQ-07 | PWA 支持（添加到主屏幕、竖屏锁定）| P1 | 体验增强 |
| REQ-08 | 横屏检测与警告提示 | P1 | 体验增强 |
| REQ-09 | 精致的页面指示器（紧凑/展开双模式）| P1 | 交互设计 |
| REQ-10 | 图片高质量加载与骨架屏 | P1 | 性能要求 |

### S-03 博物馆选择依据

| 博物馆 | 选择理由 | 特色 |
|--------|---------|------|
| 故宫博物院 | 中国最具代表性的博物馆，中文用户最高认知度 | 中国传统文化、皇家珍藏 |
| 乌菲齐美术馆 | 文艺复兴艺术殿堂，艺术爱好者必访 | 波提切利、达芬奇、拉斐尔 |
| 大英博物馆 | 世界最大综合性博物馆，藏品横跨全球文明 | 跨文明藏品、免费开放 |

## 3. 竞品分析

| 竞品 | 优势 | 不足 | 借鉴点 |
|------|------|------|--------|
| 小红书博物馆攻略 | 内容丰富、用户生产 | 碎片化、无结构、不可离线 | 垂直滑动交互模式 |
| 博物馆官方 App | 权威、全面 | 臃肿、体验差、更新慢 | 楼层导览、展品数据 |
| 马蜂窝景点攻略 | 路线规划、用户评价 | 信息密度高、移动端体验一般 | 路线推荐逻辑 |
| 每日故宫 | 设计精美、单展品深度 | 内容单一、无路线规划 | 文艺设计风格、内容深度 |

## 4. 三期开发计划来源

项目通过三个阶段逐步完成全部功能：

| 阶段 | 计划名称 | 核心交付 | 状态 |
|------|---------|---------|------|
| 一期 | 博物馆旅行攻略 WebApp 执行计划 | 基础架构 + 三个博物馆数据 + 核心页面 + 部署上线 | ✅ 已完成 |
| 二期 | iPhone 专属移动端 H5 适配计划 | 全系 iPhone 适配 + 安全区域 + 触控优化 + PWA | ✅ 已完成 |
| 三期 | 展品介绍内容升级计划 | 双版本导览系统 + 展品详情页 + 全局偏好管理 | ✅ 已完成 |
