## 3. 核心交互架构

### 3.1 CSS Scroll Snap 方案

```
snap-container (100dvh)
├── scroll-snap-type: y mandatory
├── overflow-y: auto
├── -webkit-overflow-scrolling: touch
│
├── snap-page[0] (100dvh, scroll-snap-align: start)
├── snap-page[1] (100dvh)
├── snap-page[2] (100dvh)
└── snap-page[N] (100dvh)
```

**关键决策**: 使用 CSS 原生 Scroll Snap 而非 JS 模拟，确保 60fps 滚动性能。

### 3.2 页码追踪方案

使用 `IntersectionObserver` 替代 scroll 事件监听：
- `threshold: 0.5` — 页面可见 50% 时判定为当前页
- 无需 throttle/debounce，浏览器原生调度
- 内存友好：observe/unobserve 自动管理

## 5. ADR（架构决策记录）

### ADR-001: Hash Router vs Browser Router

| 项 | 内容 |
|---|------|
| **决策** | 采用 Hash Router (`createHashRouter`) |
| **背景** | Browser Router 在 Vercel 静态托管上刷新会 404 |
| **方案对比** | Browser Router + `_redirects` 配置 vs Hash Router |
| **理由** | Hash Router 无需任何服务端配置，开箱即用，适合纯静态部署 |
| **代价** | URL 不够美观（`#/` 前缀），SEO 不友好 |
| **日期** | 2025-03 |

### ADR-002: 图片源选择

| 项 | 内容 |
|---|------|
| **决策** | 使用 Unsplash/Wikimedia 缩略图 URL |
| **背景** | 本地图片打包体积过大；Wikimedia 原图 CORS 问题 |
| **迭代历史** | 本地图片 → Wikimedia 原图（CORS 失败）→ 800px 缩略图（部分失败）→ Unsplash CDN |
| **理由** | Unsplash 跨域友好、CDN 加速、高质量图片 |
| **代价** | 依赖外部服务可用性 |
| **日期** | 2025-03 |

### ADR-003: 页面指示器交互模式

| 项 | 内容 |
|---|------|
| **决策** | 采用紧凑/展开双模式 + 自动隐藏设计 |
| **背景** | 固定显示小圆点占用空间且视觉噪音大 |
| **方案对比** | 固定小圆点 vs 可交互双模式指示器 |
| **理由** | 隐形设计减少视觉干扰，展开模式提供导航能力 |
| **代价** | 实现复杂度增加（长按、自动隐藏定时器、动画）|
| **日期** | 2025-03 |

### ADR-004: 展品内容双版本架构

| 项 | 内容 |
|---|------|
| **决策** | 数据层扩展可选字段 + Context 全局偏好 + Tab 切换 |
| **背景** | 不同用户对内容深度有不同需求 |
| **方案对比** | 路由级双页面 vs 组件级 Tab 切换 |
| **理由** | Tab 切换用户体验更流畅，Context 偏好管理简洁 |
| **代价** | 数据量翻倍（35 展品 × 2 版本内容）|
| **日期** | 2025-04 |

### ADR-005: Snap 容器定位方案

| 项 | 内容 |
|---|------|
| **决策** | 移除 `.snap-container` 的 `position: fixed`，改用正常文档流 |
| **背景** | 固定定位导致 iOS Safari 滚动异常和回弹问题 |
| **方案对比** | `position: fixed` + `overflow: auto` vs 正常流 + `height: 100dvh` |
| **理由** | 正常流避免了 iOS 橡皮筋回弹和地址栏交互冲突 |
| **代价** | 需要额外处理页面间的定位关系 |
| **日期** | 2025-03 |

## 6. 技术债务

| 编号 | 描述 | 影响 | 优先级 |
|------|------|------|--------|
| TD-01 | 无代码分割（所有页面同步加载）| 首屏加载包含未访问页面代码 | P2 |
| TD-02 | 图片无本地缓存策略 | 离线时图片不可用 | P2 |
| TD-03 | 无 Service Worker | 不支持真正离线访问 | P2 |
| TD-04 | 无错误边界组件 | 组件崩溃可能白屏 | P1 |
| TD-05 | 数据文件体积大（内联 TS）| 影响 Tree-shaking 效果 | P2 |
