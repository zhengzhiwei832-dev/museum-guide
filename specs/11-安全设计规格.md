# 11 — 安全设计规格

| 项 | 值 |
|---|---|
| 模块编号 | M1-MuseumGuide |
| 模块名称 | 博物馆漫步 WebApp |
| 文档版本 | v1.0 |
| 文档状态 | Approved |

---

> **说明**: 本项目为纯前端静态展示应用，无用户认证、无后端服务、无用户输入。安全设计聚焦于前端资源安全、外部依赖安全和 PWA 安全。

## 1. 安全目标

| 目标 | 说明 |
|------|------|
| 保护用户设备安全 | 不执行恶意代码、不请求非必要权限 |
| 资源完整性 | 确保加载的外部资源未被篡改 |
| 隐私保护 | 不收集、不存储、不传输用户个人信息 |

## 2. 攻击面分析

| 攻击面 | 风险等级 | 说明 |
|--------|---------|------|
| 用户输入 | **无** | 应用无任何用户输入入口（无表单、无搜索框、无评论）|
| XSS | **极低** | 无动态内容注入，所有内容为静态编译 |
| CSRF | **无** | 无后端 API、无状态变更请求 |
| 外部图片 | **低** | 依赖 Unsplash/Wikimedia CDN |
| 第三方依赖 | **低** | npm 依赖通过 lock 文件锁定版本 |

## 3. 前端安全措施

### 3.1 内容安全策略 (CSP)

Vercel 默认提供基础 CSP 头。推荐配置：

| 指令 | 值 | 说明 |
|------|---|------|
| default-src | 'self' | 默认只允许同源 |
| script-src | 'self' | 仅允许同源脚本 |
| style-src | 'self' 'unsafe-inline' | Tailwind 内联样式需要 |
| img-src | 'self' https://images.unsplash.com https://upload.wikimedia.org | 允许的图片源 |
| font-src | 'self' https://fonts.gstatic.com | Google Fonts |
| connect-src | 'self' | 无外部 API 连接 |

### 3.2 传输安全

| 措施 | 实现 |
|------|------|
| HTTPS 强制 | Vercel 自动提供 SSL 证书和 HTTPS 重定向 |
| HSTS | Vercel 默认启用 Strict-Transport-Security |

### 3.3 React 安全特性

| 特性 | 说明 |
|------|------|
| JSX 自动转义 | React 默认对所有 JSX 表达式进行 HTML 转义 |
| 无 dangerouslySetInnerHTML | 项目中 MUST NOT 使用此 API |
| 无 eval/Function | 项目中 MUST NOT 使用动态代码执行 |

## 4. 数据安全

### 4.1 数据分类

| 分类 | 数据 | 敏感级别 |
|------|------|---------|
| 公开内容 | 博物馆信息、展品介绍、路线数据 | 公开 |
| 用户偏好 | 导览版本选择（popular/enthusiast）| 非敏感 |

### 4.2 localStorage 安全

| 措施 | 说明 |
|------|------|
| 无 PII | MUST NOT 存储姓名、邮箱、位置等个人信息 |
| 无 Token | MUST NOT 存储认证凭证 |
| 数据最小化 | 仅存储 1 个非敏感偏好键值对 |

## 5. 外部资源安全

### 5.1 图片源白名单

| 域名 | 用途 | 协议 |
|------|------|------|
| images.unsplash.com | 展品/博物馆图片 | HTTPS |
| upload.wikimedia.org | 部分展品图片 | HTTPS |
| fonts.googleapis.com | 字体样式表 | HTTPS |
| fonts.gstatic.com | 字体文件 | HTTPS |

### 5.2 依赖安全

| 措施 | 说明 |
|------|------|
| package-lock.json | 锁定依赖版本，防止供应链攻击 |
| 定期审计 | SHOULD 定期执行 `npm audit` |
| 最小依赖 | 生产依赖仅 4 个（react, react-dom, react-router-dom, framer-motion）|

## 6. PWA 安全

| 措施 | 说明 |
|------|------|
| manifest scope | 限制在应用根目录 `/` |
| start_url | 固定为 `/`，防止深度链接注入 |
| display: standalone | 隐藏浏览器地址栏，减少钓鱼风险提示需求 |

## 7. 部署安全

| 措施 | 说明 |
|------|------|
| Vercel 自动 HTTPS | 所有流量加密传输 |
| 不可变部署 | 每次部署生成独立 URL，支持快速回滚 |
| 无服务端代码 | 纯静态产物，无服务端注入风险 |
| 构建环境隔离 | Vercel CI/CD 在隔离容器中构建 |
