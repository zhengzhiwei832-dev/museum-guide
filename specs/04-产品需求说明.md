## 1. 愿景与目标

### 1.1 产品愿景

打造一款「让逛博物馆像刷短视频一样上瘾」的移动端导览体验。用文艺的设计语言、沉浸式的交互方式、分层的深度内容，让每一位用户都能找到属于自己的博物馆打开方式。

### 1.2 痛点→目标映射

| 痛点 | 目标 |
|------|------|
| 攻略碎片化 | 提供三个世界级博物馆的结构化完整导览 |
| 移动端体验差 | 全屏吸附滑动 + iPhone 专属适配 |
| 内容深度单一 | 大众版/爱好者版双版本内容体系 |
| 现场不知看什么 | 智能路线推荐 + 时间感知 |
| 不支持离线 | PWA 添加到主屏幕 |

## 2. 目标用户

| 角色 | 代号 | 使用场景 | 核心价值 |
|------|------|---------|---------|
| 旅行规划者 | U-01 | 出行前在家做功课 | 快速了解博物馆概况、规划路线 |
| 现场自由游客 | U-02 | 博物馆内自由探索 | 随走随看、了解展品信息 |
| 现场定时游客 | U-03 | 博物馆内有限时间 | 按时间获取最优路线方案 |
| 艺术爱好者 | U-04 | 任何场景深度阅读 | 获取学术级展品解读 |

## 3. 核心业务场景

### SC-01 首页浏览与博物馆选择

用户打开应用，看到精美的首页和博物馆列表，选择感兴趣的博物馆进入。

**流程**: 打开首页 → 浏览博物馆卡片 → 点击进入博物馆

### SC-02 提前做攻略

用户选择「我在做攻略」模式，通过垂直滑动依次浏览博物馆简介、实用信息（票价/交通/注意事项）、楼层导览、代表性展品和冷门展品。

**流程**: 选择攻略模式 → 上滑浏览封面 → 实用信息 → 楼层导览 → 展品卡片 → 攻略总结

### SC-03 现场自由探索

用户已在博物馆现场，选择「自由探索」模式，直接浏览展品信息，重点展示位置信息便于寻找。

**流程**: 选择在现场 → 自由探索 → 上滑浏览楼层 → 展品卡片

### SC-04 智能路线推荐

用户在现场但时间有限，选择「推荐路线」模式，先选择可用时间（1h/2h/半天/全天），系统推荐对应路线方案。

**流程**: 选择在现场 → 推荐路线 → 选择时间 → 展示路线站点 → 按站点游览

### SC-05 展品深度导览

用户对某件展品感兴趣，点击进入展品详情页，可切换大众版/爱好者版查看不同深度的解读内容。

**流程**: 展品卡片点击「详细介绍」→ 展品详情页 → 切换版本 Tab → 阅读内容

## 4. 产品规则

| 编号 | 规则 | 说明 |
|------|------|------|
| R-01 | 所有页面 MUST 为竖屏全屏布局 | 使用 100dvh 动态视口高度 |
| R-02 | 滑动 MUST 有吸附效果 | CSS Scroll Snap mandatory |
| R-03 | 展品图片 MUST 懒加载 | loading="lazy" + decoding="async" |
| R-04 | 页面指示器 MUST 支持双模式 | 紧凑模式（进度条）+ 展开模式（目录）|
| R-05 | 路线推荐 MUST 按时间分组 | 至少覆盖 1h/2h/半天/全天 |
| R-06 | 展品详情 MUST 支持版本切换 | 大众版和爱好者版独立内容 |
| R-07 | 版本偏好 MUST 持久化 | localStorage 存储用户选择 |
| R-08 | 横屏时 MUST 显示提示 | 引导用户切回竖屏 |
| R-09 | 导航返回 MUST 可点击 | 所有子页面有返回按钮 |
| R-10 | 触控目标 MUST ≥ 44px | 符合 Apple HIG |
